The exchange onramp lets users initiate a transfer directly from their CEX account (Coinbase, Binance, Robinhood, and others) without leaving your app. The OAuth connection and transfer are handled automatically — Trails receives the funds and routes them to the destination.

No separate account or additional API keys are required. Trails manages the exchange session server-side.

## Prerequisites

- A Trails API key from [dashboard.trails.build](https://dashboard.trails.build)
- The `0xtrails` SDK installed

## Setup

### 1. Add the `onramp` prop to `Fund`

Pass an `onramp` config object with the environment set:

```tsx
import { Fund } from '0xtrails/widget'

<Fund
  apiKey={process.env.NEXT_PUBLIC_TRAILS_API_KEY}
  onramp={{
    mesh: {
      environment: 'production',
    },
  }}
/>
```

This enables the "Fund from exchange" option inside the fund flow. Users will see a list of available exchanges to connect.

### 2. Pre-select the exchange flow

To skip the funding method selector and open the exchange flow directly, pass `paymentMethod="EXCHANGE"`:

```tsx
<Fund
  apiKey={process.env.NEXT_PUBLIC_TRAILS_API_KEY}
  paymentMethod="EXCHANGE"
  onramp={{
    mesh: {
      environment: 'production',
    },
  }}
/>
```

### 3. Restrict to specific exchanges

The `exchanges` array filters which exchanges are shown. If omitted, all supported exchanges are listed.

```tsx
<Fund
  apiKey={process.env.NEXT_PUBLIC_TRAILS_API_KEY}
  onramp={{
    mesh: {
      environment: 'production',
      exchanges: ['coinbase', 'robinhoodDirect'],
    },
  }}
/>
```

## Using sandbox mode

During development, use `environment: 'sandbox'` to test the flow without real funds. A yellow banner is shown inside the widget when sandbox is active.

```tsx
<Fund
  apiKey={process.env.NEXT_PUBLIC_TRAILS_API_KEY}
  onramp={{
    mesh: {
      environment: 'sandbox',
    },
  }}
/>
```

## Full example

```tsx
import { Fund } from '0xtrails/widget'

export function App() {
  return <FundWithExchange />
}

function FundWithExchange() {
  return (
    <Fund
      apiKey={process.env.NEXT_PUBLIC_TRAILS_API_KEY}
      to={{
        recipient: "0xYourRecipientAddress",
        currency: "USDC",
        chain: "base",
      }}
      paymentMethod="EXCHANGE"
      onramp={{
        mesh: {
          environment: 'production',
        },
      }}
      onFundingSuccess={({ sessionId }) => {
        console.log('Exchange transfer initiated:', sessionId)
      }}
    />
  )
}
```

## How the auth flow works

1. The user selects an exchange from the list.
2. An OAuth popup or redirect opens to the exchange's login.
3. After authentication, a transfer confirmation is shown with the amount, token, and destination.
4. The user confirms and the withdrawal is initiated from their exchange account.
5. Once the transfer is detected on-chain, Trails routes it to the destination as configured.

The widget shows a pending state while the exchange processes the withdrawal. Session tokens are stored locally for 5 minutes so the user does not have to re-authenticate if they retry.

## Supported exchanges

Available exchanges are returned dynamically. The current list includes Coinbase, Binance, Robinhood, and others.

## `fundMethodsList` integration

The exchange flow can be combined with `fundMethodsList` to control what funding methods appear in the selector:

```tsx
<Fund
  apiKey={process.env.NEXT_PUBLIC_TRAILS_API_KEY}
  onramp={{
    mesh: {
      environment: 'production',
    },
  }}
  fundMethodsList={['wallet', 'direct-transfer', 'onramp-mesh']}
  hideUnlistedFundMethods={true}
/>
```

Accepted values in `fundMethodsList`: `"wallet"`, `"direct-transfer"`, `"onramp-mesh"`, `"onramp-meld"`.

## Troubleshooting

**Exchange list is empty**: Confirm `apiKey` is set correctly. The exchange list fetch requires a valid API key.

**Popup blocked**: The exchange connection opens a popup window. If the browser blocks it, the widget shows a "Retry" button. Trigger the flow from a user gesture (button click) rather than programmatically, and encourage users to allow popups for your domain.

**Session expired**: The exchange session is valid for 5 minutes. If it expires before the user completes the transfer, a "Refresh Session" button appears to restart the flow.

**Sandbox not showing real exchanges**: In sandbox mode, test integrations are returned. Switch to `environment: 'production'` to see real exchange options.
